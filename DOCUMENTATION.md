# Binoculars AI Text Detection — Experiment Documentation

## 1. Background

### Problem

NIH compliance concerns require awareness of AI-generated content in grant applications. The **Grant Assist** project needs a mechanism to detect whether submitted text was likely generated by an LLM.

### Approach

**Binoculars** (ICML 2024, [arxiv.org/abs/2401.12070](https://arxiv.org/abs/2401.12070)) is a zero-shot AI text detector that compares two related LLMs on the same text. It requires no training data or fine-tuning.

The score formula is:

```
score = PPL_observer(text) / X-PPL_observer,performer(text)
```

- **PPL** = perplexity: how surprised the observer model is by the actual tokens
- **X-PPL** = cross-perplexity: how much the observer and performer models' full distributions disagree

The paper claims: AI text → score ≈ 1.0 (both models agree), human text → score < 1.0 (models diverge).

### Why Not Ollama?

Binoculars requires **full logit distributions** over the entire vocabulary (~65K tokens) from both models. Ollama only exposes the top-20 logprobs via its API. This is a hard constraint — the cross-perplexity calculation sums over all vocabulary tokens. We use the HuggingFace `transformers` library instead, which provides full logit access.

## 2. Implementation

### Platform

- **Mac Studio** with Apple Silicon (MPS backend)
- PyTorch float16 (no quantization — BitsAndBytes not supported on MPS)
- ~28-32 GB unified memory required for two 7B models

### Adaptation from Cloud Run Version

The Grant Assist repo (`grant_assist/binoculars-service/`) contains a CUDA-based implementation for GCP Cloud Run with L4 GPU. We adapted it for Mac Studio:

| Change | Cloud Run | Mac Studio |
|---|---|---|
| Device | CUDA | MPS (Apple Metal) |
| Quantization | 4-bit (BitsAndBytes) | None (float16) |
| Memory | ~8 GB (quantized) | ~28 GB (fp16) |
| Numerical stability | Default | Cast to float32 for cross_entropy on MPS |
| Deployment | Docker + Cloud Run | Local Python process |

The API contract (`POST /detect`) is identical — the local server is a drop-in replacement.

## 3. Experiments Conducted

### Experiment 1: Falcon-7B on Original Samples

**Goal:** Baseline test of Binoculars with the paper's recommended model pair.

**Models:** `tiiuae/falcon-7b` (observer) + `tiiuae/falcon-7b-instruct` (performer)

**Samples:** 17 total — 5 human-written grant sections (with personal voice, lab anecdotes, informal asides), 5 AI-generated grant sections (smooth, formulaic, polished), 3 mixed, 4 edge cases.

**Result:**
- Human mean score: 0.72, AI mean score: 0.60
- All scores below the published thresholds (0.85/0.90)
- AI text scored **lower** than human text — opposite of paper's prediction
- 0% AI detection rate with published thresholds

**Script:** `scripts/run_test_samples.py`

---

### Experiment 2: Cross-Architecture AI Text via Ollama

**Goal:** Test whether the score inversion is specific to Claude-generated text, or affects all AI model families.

**Models (text generation):** Qwen3:8B, Gemma3:12B, Llama3.1:8B via Ollama
**Models (detection):** Falcon-7B pair (same as Experiment 1)

**Samples:** 6 grant sections generated by 3 different LLM architectures.

**Result:**
- Ollama AI mean: 0.63, Claude AI mean: 0.60, Human mean: 0.72
- All AI sources scored lower than human text
- The inversion is **architecture-independent** — not a Claude-specific artifact

**Script:** `scripts/run_generated_samples.py`

---

### Experiment 3: Qwen2.5-7B Alternative Model Pair

**Goal:** Test whether a newer model pair produces better separation or correct score direction.

**Models:** `Qwen/Qwen2.5-7B` (observer) + `Qwen/Qwen2.5-7B-Instruct` (performer)

**Result:**
- Scores shifted dramatically higher (0.81–1.10)
- But direction remained **wrong**: human mean 1.04 > AI mean 0.89
- Accuracy dropped to 9% — worse than Falcon-7B
- The inversion is not specific to the Falcon model family

**Script:** `scripts/run_alt_model.py --model-pair qwen2.5-7b`

---

### Experiment 4: Threshold Calibration on Expanded Samples

**Goal:** Determine whether grant-specific thresholds can salvage detection accuracy.

**Samples:** 30 labeled samples (15 human, 15 AI) — original 10 + 20 new from `test_samples_expanded.py`.

**Statistical Tests:**
- Mann-Whitney U: p = 0.00008 (significant)
- Welch's t-test: p = 0.000002 (significant)
- Cohen's d = 2.23 (large effect size)

**Calibrated Thresholds (inverted — lower = AI):**

| Method | Threshold | Sensitivity | FPR | Accuracy |
|---|---|---|---|---|
| Youden's J (optimal) | 0.6845 | 93.3% | 20.0% | 86.7% |
| Conservative (≤1% FPR) | 0.6387 | 60.0% | 0% | 80.0% |

**Script:** `scripts/calibrate_thresholds.py`

---

### Experiment 5: Score Component Decomposition

**Goal:** Understand *why* the scores invert — decompose into PPL and X-PPL.

**Result:**

| Component | Human | AI | Change |
|---|---|---|---|
| PPL (observer) | 2.24 | 1.51 | **-32.8%** |
| X-PPL (cross) | 3.13 | 2.49 | -20.5% |
| KL divergence | 74.2 | 54.6 | -26.4% |
| Score (PPL/X-PPL) | 0.72 | 0.60 | -16.5% |

**Explanation:** AI-generated grant text is extremely predictable to the observer model (PPL drops 33%). The cross-perplexity (X-PPL) also drops but has a natural floor set by the inherent divergence between base and instruct model distributions — it can only drop 20%. Since the numerator (PPL) shrinks faster than the denominator (X-PPL), the ratio decreases rather than increasing.

The KL divergence confirms the paper's core theory: models **do** agree more on AI text (KL = 54.6 vs 74.2). The theory is correct; the ratio formulation just breaks when PPL is driven very low by domain-specific predictability.

**Script:** `scripts/diagnose_scores.py`

## 4. Key Findings

### Finding 1: Binoculars CAN detect AI grant text

Statistical separation is significant (p < 0.0001, Cohen's d = 2.23). With calibrated thresholds, 87% accuracy is achievable. Detection works — it just requires domain-specific calibration.

### Finding 2: Score direction is inverted

On grant text, AI scores **lower** than human text. This is the opposite of the paper's prediction but is mathematically explained by the asymmetric drop in PPL vs X-PPL. The detection rule must be flipped: `score < threshold → AI`.

### Finding 3: Published thresholds are useless for this domain

All 37 samples scored below 0.85. The published thresholds (0.8536/0.9015) achieve 0% detection. Grant-specific thresholds must be used.

### Finding 4: The inversion is universal

Tested across 4 AI text generators (Claude, Qwen3, Gemma3, Llama3.1) and 2 detector model pairs (Falcon-7B, Qwen2.5-7B). The inversion holds in every combination.

### Finding 5: There is an overlap zone

Scores between ~0.645 and ~0.710 contain both human and AI samples. Samples in this range should be classified as "uncertain" rather than making a definitive call.

---

## 5. Recommended Grant-Specific Thresholds

```
score < 0.64  →  ai_generated   (high confidence, ~0% FPR)
0.64 ≤ score < 0.69  →  uncertain  (overlap zone)
score ≥ 0.69  →  human          (high confidence)
```

These thresholds are calibrated on 30 samples and should be validated on a larger corpus of real grant documents before production use.

---

## 6. Caveats and Limitations

1. **Sample size:** 30 labeled samples is a starting point, not a production-ready calibration. Need 100+ real grant sections.
2. **All AI samples were synthetically generated** by Claude and local LLMs. Real-world AI usage in grants may differ (e.g., AI-assisted editing vs. full AI generation).
3. **English only.** Binoculars degrades on non-English text.
4. **512 token limit** per chunk. Long documents must be chunked.
5. **The "AI" samples are written to be stereotypically AI-like.** A sophisticated user who deliberately introduces imperfections would evade detection.
6. **Falcon-7B is from 2023.** As AI writing evolves, detection may degrade. Periodic recalibration is needed.

## 7. Next Steps

1. **Validate on real grant data** — score actual human-written grants from existing Grant Assist reviews alongside AI-rewritten versions of the same grants.
2. **Ensemble with stylometric features** — sentence length variance, vocabulary richness, burstiness, and other statistical measures as complementary signals.
3. **Integrate into Grant Assist** — update `backend/app/services/ai_detection.py` to use inverted thresholds and the local server for development.
4. **Investigate domain-specific model pairs** — a pair fine-tuned on biomedical text may produce better separation.
5. **Test on non-academic text** — confirm whether the inversion is specific to formal/academic domains or a broader phenomenon.
